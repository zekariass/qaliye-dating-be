# Frontend/Mobile Fix: Discovery Actions Request Body

## Problem
Discovery actions (like, pass, superlike) are failing with validation errors because the request body is missing required fields.

## Error Log
```
Field error in object 'swipeActionRequest' on field 'clientActionId': rejected value [null]; codes [NotNull.swipeActionRequest.clientActionId,NotNull.clientActionId,NotNull.java.util.UUID,NotNull]; default message [must not be null]
Field error in object 'swipeActionRequest' on field 'targetUserId': rejected value [null]; codes [NotNull.swipeActionRequest.targetUserId,NotNull.targetUserId,NotNull.java.util.UUID,NotNull]; default message [must not be null]
```

## API Endpoints Affected
- `POST /api/v1/discovery/actions/like`
- `POST /api/v1/discovery/actions/pass`
- `POST /api/v1/discovery/actions/superlike`

## Required Request Body Format
All three endpoints expect a JSON body with these fields:

```json
{
  "targetUserId": "uuid-of-target-user",
  "clientActionId": "uuid-generated-by-client"
}
```

## Field Descriptions
- **targetUserId** (UUID, required): The ID of the profile being liked/passed/superliked
- **clientActionId** (UUID, required): A unique ID generated by the client for idempotency (prevents duplicate actions if the request is retried)

## Fix Required
Update the client code to:
1. Include the `targetUserId` from the profile being acted upon
2. Generate a unique `clientActionId` (UUID) for each action request before sending
3. Send both fields in the request body as JSON

## Example Implementation
```javascript
// Before making the API call
const clientActionId = crypto.randomUUID(); // or equivalent UUID generator

const requestBody = {
  targetUserId: profile.id,  // The profile being liked/passed
  clientActionId: clientActionId
};

// Then send to endpoint
fetch('/api/v1/discovery/actions/like', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify(requestBody)
});
```

## Note
The `rewind` endpoint (`POST /api/v1/discovery/actions/rewind`) does NOT require a request body - it only needs the JWT authentication.
